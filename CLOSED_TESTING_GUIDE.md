# 📱 Weekly Gacha - 비공개 테스트 가이드

## 🎯 목표

Google Play Console에서 **비공개 테스트**를 진행하여 프로덕션 출시 권한을 획득합니다.

---

## ✅ **필수 요구사항**

- ✅ **테스터 수**: 최소 12명
- ✅ **테스트 기간**: 최소 14일 (연속)
- ✅ **테스터 상태**: 14일 동안 "참여" 상태 유지

---

## 📋 **1단계: Google Play Console에서 비공개 테스트 설정**

### A. 비공개 테스트 트랙 만들기

1. **Google Play Console 접속**
   - https://play.google.com/console

2. **앱 선택** → **테스트 및 출시** → **테스트** → **비공개 테스트**

3. **출시 만들기** 클릭

4. **AAB 파일 업로드**
   - 파일: `app-release.aab` (77MB)
   - 버전: v2.8.5 (Build 22)

5. **출시 이름 입력**
   ```
   Weekly Gacha v2.8.5 - 비공개 테스트
   ```

6. **출시 정보 입력**
   ```
   초대 코드 기능 개선
   - 대소문자 구분 없이 작동
   - 상세한 에러 메시지 추가
   - 디버깅 로그 강화
   ```

---

### B. 테스터 그룹 만들기

1. **테스터 탭** 클릭

2. **새 목록 만들기**
   - 목록 이름: `Weekly Gacha 베타 테스터`

3. **테스터 추가 방법 선택**

**방법 1: 이메일 주소로 추가**
```
tester1@gmail.com
tester2@gmail.com
tester3@gmail.com
...
(최소 12명의 이메일 주소)
```

**방법 2: Google 그룹 사용**
- Google Groups에서 그룹 생성
- 그룹 이메일 주소 입력

---

## 👥 **2단계: 테스터 모집 방법**

### 추천 테스터 모집 경로

**1. 가족 및 친구 (가장 쉬움)**
- 최소 12명 모집
- 지속적으로 앱 사용 가능한 사람
- 14일 동안 참여 상태 유지 약속

**2. 직장 동료**
- IT 관련 직종이면 더 좋음
- 앱 테스트에 관심 있는 동료

**3. 온라인 커뮤니티**
- 게임 관련 커뮤니티
- 개발자 커뮤니티 (카카오톡, 디스코드, 페이스북 그룹)

**4. SNS 활용**
- 인스타그램, 트위터, 페이스북에 테스터 모집 게시
- "가챠 게임 테스터 모집" 해시태그 활용

---

## 📧 **3단계: 테스터에게 초대 링크 전송**

### A. 초대 링크 생성

1. Play Console → 비공개 테스트
2. **테스터** 탭 → **공유 가능한 링크 복사**
3. 링크 형식:
   ```
   https://play.google.com/apps/testing/com.mycompany.weeklygacha
   ```

### B. 테스터에게 보낼 메시지 템플릿

```
안녕하세요!

Weekly Gacha 앱의 베타 테스터로 초대합니다 🎉

📱 앱 소개:
매주 새로운 테마의 디지털 포토카드를 무료로 수집하는 가챠 게임입니다.

🎯 테스터 참여 방법:
1. 아래 링크를 클릭하여 베타 테스터 등록
2. Google Play에서 앱 다운로드
3. 최소 14일 동안 앱 사용 및 피드백 제공

🔗 베타 테스터 등록 링크:
https://play.google.com/apps/testing/com.mycompany.weeklygacha

⚠️ 중요 사항:
- 등록 후 최소 14일 동안 "참여" 상태를 유지해주세요
- 앱을 자주 사용하지 않아도 되지만, 테스터 목록에서 나가지 말아주세요
- 버그나 개선사항이 있으면 알려주세요!

🎁 감사의 표시:
- 정식 출시 후 특별 보너스 티켓 제공
- 앱 개선에 큰 도움이 됩니다

참여해주셔서 감사합니다!
```

---

## 📊 **4단계: 테스터 관리 및 피드백 수집**

### A. 테스터 참여 상태 확인

**매일 확인 항목:**
1. Play Console → 비공개 테스트 → 통계
2. **현재 활성 테스터 수** 확인
3. 최소 12명 유지 여부 확인

**중요**: 테스터가 14일 동안 계속 참여해야 하므로, 중간에 나가는 사람이 있으면 추가 모집 필요!

### B. 피드백 수집

**Play Console에서 확인:**
- 평점 및 리뷰 → 테스트 의견
- 테스터들의 비공개 의견 확인

**추가 피드백 채널:**
- 이메일 수신 주소 제공
- 카카오톡 오픈채팅방 개설
- Google Forms 설문지 작성

### C. 피드백 기록 템플릿

```
[날짜] [테스터 이름] [의견 내용]
2024-12-11  테스터1  가챠 애니메이션이 부드럽지 않음
2024-12-12  테스터2  초대 코드 기능 잘 작동함
2024-12-13  테스터3  컬렉션 화면 로딩이 느림
...
```

## 📝 **6단계: 프로덕션 액세스 신청**

### A. 신청 조건 확인

**체크리스트:**
- [ ] 12명 이상의 테스터가 등록
- [ ] 14일 동안 지속적으로 참여 상태 유지
- [ ] 피드백 수집 및 기록 완료
- [ ] 주요 버그 수정 완료

### B. 신청 방법

1. **대시보드** → **프로덕션 신청** 클릭

2. **질문에 답변** (3개 섹션)

**섹션 1: 비공개 테스트 정보**
```
Q: 비공개 테스트에 몇 명이 참여했나요?
A: 12명 (또는 실제 참여 인원)

Q: 테스트 기간은 얼마나 되었나요?
A: 14일

Q: 테스터로부터 받은 주요 피드백은 무엇인가요?
A: [피드백 요약]
예시:
- 가챠 애니메이션 개선 요청 → 속도 최적화 완료
- 초대 코드 기능 버그 발견 → v2.8.5에서 수정 완료
- UI/UX 개선 제안 → 일부 적용, 나머지는 향후 업데이트 예정
```

**섹션 2: 앱/게임 정보**
```
Q: 앱의 주요 기능은 무엇인가요?
A: 매주 새로운 테마의 디지털 포토카드를 무료로 수집하는 가챠 게임

Q: 타겟 사용자는 누구인가요?
A: 수집형 게임을 좋아하는 전 연령대 사용자

Q: 앱이 Google Play 정책을 준수하나요?
A: 예, 개인정보방침 제공, AdMob 광고 정책 준수
```

**섹션 3: 프로덕션 준비 상태**
```
Q: 앱이 프로덕션 출시 준비가 되었나요?
A: 예

Q: 주요 버그가 모두 수정되었나요?
A: 예, 테스트 기간 중 발견된 버그 모두 수정 완료

Q: 앱 스토어 등록정보가 완성되었나요?
A: 예, 스크린샷, 기능 그래픽, 앱 설명 모두 준비 완료
```

### C. 신청 후

- 구글 검토 기간: **약 7일**
- 결과는 이메일로 통보
- 승인되면 **프로덕션 트랙 사용 가능**
